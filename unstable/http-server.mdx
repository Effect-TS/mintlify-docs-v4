## Overview

The `effect/unstable/http` modules provide tools for building HTTP servers with
routing, middleware, and typed request handling.

The router is built up as a `Layer` that requires the `HttpRouter` service. The
recommended entry point is `HttpRouter.use`: it gives you direct access to the
router so you can register every route and middleware inside one effectful
block. You then hand the combined application layer to `HttpRouter.serve` (a
long-running server) or `HttpRouter.toWebHandler` (a serverless `fetch`-style
handler).

## Creating a server

```typescript
import { NodeHttpServer, NodeRuntime } from "@effect/platform-node"
import { Effect, Layer } from "effect"
import { HttpRouter, HttpServerResponse } from "effect/unstable/http"
import { createServer } from "node:http"

const AppLive = HttpRouter.use((router) =>
  router.add("GET", "/health", HttpServerResponse.text("ok"))
)

const ServerLive = HttpRouter.serve(AppLive).pipe(
  Layer.provide(NodeHttpServer.layer(createServer, { port: 3000 }))
)

Layer.launch(ServerLive).pipe(NodeRuntime.runMain)
```

## Routing

### Registering routes with `HttpRouter.use`

`HttpRouter.use((router) => Effect)` returns a `Layer` whose body runs with the
router in scope. Call `router.add(method, path, handler)` for each route — a
handler can be an `HttpServerResponse` value, an `Effect` producing one, or a
function from the request to such an effect.

```typescript
import { Effect } from "effect"
import { HttpRouter, HttpServerResponse } from "effect/unstable/http"

const AppLive = HttpRouter.use((router) =>
  Effect.gen(function*() {
    yield* router.add(
      "GET",
      "/",
      HttpServerResponse.text("Welcome")
    )

    yield* router.add(
      "POST",
      "/echo",
      (request) =>
        Effect.map(request.text, (body) => HttpServerResponse.text(body))
    )

    yield* router.add(
      "GET",
      "/users/:id",
      Effect.gen(function*() {
        const params = yield* HttpRouter.params
        return HttpServerResponse.text(`user ${params.id}`)
      })
    )
  })
)
```

Path segments prefixed with `:` are captured and exposed by the
`HttpRouter.params` effect inside any handler.

### Grouping routes with a prefix

`router.prefixed(prefix)` returns a router whose subsequent `add` calls are
mounted under that prefix:

```typescript
import { Effect } from "effect"
import { HttpRouter, HttpServerResponse } from "effect/unstable/http"

const AppLive = HttpRouter.use((router) =>
  Effect.gen(function*() {
    const api = router.prefixed("/api")

    yield* api.add(
      "GET",
      "/users",
      HttpServerResponse.text("users")
    )

    yield* api.add(
      "POST",
      "/users",
      HttpServerResponse.text("created")
    )
  })
)
```

### Schema-decoded inputs

`HttpRouter.schemaJson` decodes the entire request (method, URL, headers,
cookies, path params, search params, and JSON body) against a schema in one
step. `HttpRouter.schemaPathParams` and `HttpRouter.schemaNoBody` cover the
non-body cases.

```typescript
import { Effect, Schema } from "effect"
import { HttpRouter, HttpServerResponse } from "effect/unstable/http"

const CreateUser = Schema.Struct({
  body: Schema.Struct({
    name: Schema.String,
    age: Schema.Number
  })
})

const AppLive = HttpRouter.use((router) =>
  router.add(
    "POST",
    "/users",
    Effect.gen(function*() {
      const { body } = yield* HttpRouter.schemaJson(CreateUser)
      return HttpServerResponse.text(`created ${body.name}`)
    })
  )
)
```

## Request handling

### Accessing the request

Inside a handler, access the request via the `HttpServerRequest` service (or
through the function form, which receives it as an argument):

```typescript
import { Effect } from "effect"
import { HttpServerRequest, HttpServerResponse } from "effect/unstable/http"

const handler = Effect.gen(function*() {
  const request = yield* HttpServerRequest.HttpServerRequest
  const body = yield* request.json
  return HttpServerResponse.text(JSON.stringify(body))
})
```

`request.json`, `request.text`, `request.arrayBuffer`, and `request.stream`
read the body. `request.cookies`, `request.headers`, `request.method`, and
`request.url` expose request metadata.

### Sending responses

`HttpServerResponse` provides constructors for the common cases. `json` and
`html` (when used as a template tag) return effects because they can fail;
`text`, `empty`, and `raw` are synchronous values.

```typescript
import { Effect } from "effect"
import { HttpServerResponse } from "effect/unstable/http"

// Text response
const textHandler = HttpServerResponse.text("Hello World")

// JSON response — returns an Effect because serialization can fail
const jsonHandler = HttpServerResponse.json({ message: "Success" })

// Empty response with a custom status
const createdHandler = Effect.succeed(
  HttpServerResponse.empty({ status: 201 })
)
```

If you have a known-serializable value and don't want the error in your
signature, use `HttpServerResponse.jsonUnsafe`.

## Middleware

### Defining middleware

`HttpMiddleware.make` wraps the per-request handler effect. You can inspect the
request, run the inner handler, and modify the response.

```typescript
import { Effect } from "effect"
import { HttpMiddleware, HttpServerRequest } from "effect/unstable/http"

const Logger = HttpMiddleware.make((app) =>
  Effect.gen(function*() {
    const request = yield* HttpServerRequest.HttpServerRequest
    yield* Effect.log(`${request.method} ${request.url}`)
    return yield* app
  })
)
```

### Applying middleware

Per-route middleware is provided to the route layer with
`HttpRouter.middleware(fn).layer`. For middleware that should run on every
request, register it from inside `HttpRouter.use` via
`router.addGlobalMiddleware(fn)`.

```typescript
import { Effect, Layer } from "effect"
import { HttpRouter, HttpServerResponse } from "effect/unstable/http"

// Apply Logger to every route in the router
const AppLive = HttpRouter.use((router) =>
  Effect.gen(function*() {
    yield* router.addGlobalMiddleware(Logger)

    yield* router.add("GET", "/", HttpServerResponse.text("ok"))
  })
)

// Or scope a middleware layer to a subset of routes
const Protected = HttpRouter.use((router) =>
  router.add("GET", "/admin", HttpServerResponse.text("secret"))
).pipe(
  Layer.provide(HttpRouter.middleware(Logger).layer)
)
```

### Built-in middleware

The package ships with a few ready-made middleware layers:

- `HttpRouter.cors(options?)` — applies CORS headers globally
- `HttpRouter.disableLogger` — silences the request logger for a subset of routes
- `HttpRouter.provideRequest(layer)` — provides per-request services to a route group

```typescript
import { Layer } from "effect"
import { HttpRouter } from "effect/unstable/http"

const AppLayer = Layer.mergeAll(
  AppLive,
  HttpRouter.cors({ allowedOrigins: ["https://example.com"] })
)
```

## Platform support

### Node.js

```typescript
import { NodeHttpServer } from "@effect/platform-node"
import { createServer } from "node:http"

const ServerLayer = NodeHttpServer.layer(createServer, { port: 3000 })
```

`NodeHttpServer.layerConfig(...)` reads the listen options from `Config`, and
`NodeHttpServer.layerTest` provides an ephemeral-port server for integration
tests.

### Bun

```typescript
import { BunHttpServer } from "@effect/platform-bun"

const ServerLayer = BunHttpServer.layer({ port: 3000 })
```

## Web handler

For serverless and edge runtimes, `HttpRouter.toWebHandler` returns a standard
`(request: Request) => Promise<Response>` function plus a `dispose` for
cleanup:

```typescript
import { HttpRouter } from "effect/unstable/http"

const { handler, dispose } = HttpRouter.toWebHandler(AppLive, {
  disableLogger: true
})

// Use `handler` as the default export of your serverless function
export default handler
```
