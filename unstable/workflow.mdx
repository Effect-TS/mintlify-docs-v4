## Overview

The `effect/unstable/workflow` modules provide tools for building durable workflows that:

- Survive process restarts
- Handle long-running operations
- Maintain state across interruptions
- Coordinate distributed activities

## What are workflows?

Workflows are durable execution contexts that can:

- **Persist state** — workflow progress is stored and restored across restarts
- **Handle delays** — wait for hours, days, or weeks without blocking
- **Coordinate activities** — orchestrate multiple operations across services
- **Recover from failures** — automatically retry failed activities

## Defining a workflow

A workflow is defined with `Workflow.make(tag, options)`. The execution body
is attached separately via `workflow.toLayer(execute)`. This mirrors how
[`Rpc.make`](/unstable/rpc) is structured.

```typescript
import { Effect, Schema } from "effect"
import { Workflow } from "effect/unstable/workflow"

const IncrementWorkflow = Workflow.make("IncrementWorkflow", {
  payload: { value: Schema.Number },
  success: Schema.Number,
  idempotencyKey: ({ value }) => String(value)
})

const IncrementWorkflowLayer = IncrementWorkflow.toLayer(({ value }) =>
  Effect.succeed(value + 1)
)
```

`Workflow.make` accepts these options:

- `payload` — a `Schema.Struct` field map (or a struct schema) describing the workflow input
- `idempotencyKey` — pure function from the payload to a deterministic key used to derive the workflow's execution ID
- `success` *(optional, default `Schema.Void`)* — schema for the workflow result
- `error` *(optional, default `Schema.Never`)* — schema for typed workflow failures
- `suspendedRetrySchedule` *(optional)* — schedule used when a suspended workflow is resumed
- `annotations` *(optional)* — a `Context.Context<never>` of annotations

## Class-based workflows

You can extend `Workflow.make` to get a named class with the same interface:

```typescript
import { Effect, Schema } from "effect"
import { Workflow } from "effect/unstable/workflow"

class IncrementWorkflow extends Workflow.make("IncrementWorkflow", {
  payload: { value: Schema.Number },
  success: Schema.Number,
  idempotencyKey: ({ value }) => String(value)
}) {}

const IncrementWorkflowLayer = IncrementWorkflow.toLayer(({ value }) =>
  Effect.succeed(value + 1)
)
```

## Executing workflows

Call `workflow.execute(payload)` to run the workflow and await its result, or
pass `{ discard: true }` to fire-and-forget and receive just the deterministic
execution ID:

```typescript
import { Effect, Layer } from "effect"
import { WorkflowEngine } from "effect/unstable/workflow"

const program = Effect.gen(function*() {
  // Await the result — typed as the workflow's success schema
  const result = yield* IncrementWorkflow.execute({ value: 1 })

  // Fire-and-forget — returns the deterministic execution ID
  const executionId = yield* IncrementWorkflow.execute(
    { value: 1 },
    { discard: true }
  )

  return { result, executionId }
}).pipe(
  Effect.provide(IncrementWorkflowLayer.pipe(
    Layer.provideMerge(WorkflowEngine.layerMemory)
  ))
)
```

`WorkflowEngine.layerMemory` is an in-memory engine that's handy for tests and
development. Production deployments wire a persistent engine in its place.

## Polling and execution IDs

You can derive an execution ID up front, then poll for the workflow's outcome:

```typescript
import { Effect, Option } from "effect"
import { Workflow } from "effect/unstable/workflow"

const program = Effect.gen(function*() {
  const executionId = yield* IncrementWorkflow.executionId({ value: 1 })

  yield* IncrementWorkflow.execute({ value: 1 }, { discard: true })

  const polled = yield* IncrementWorkflow.poll(executionId)
  if (Option.isSome(polled) && polled.value._tag === "Complete") {
    return polled.value.exit
  }
  return null
})
```

## Activities

`Activity.make` wraps an effect so the engine can persist its result and
replay it on subsequent runs. Activities are defined with an options object:

```typescript
import { Effect, Schema } from "effect"
import { Activity } from "effect/unstable/workflow"

const reserveInventory = Activity.make({
  name: "reserveInventory",
  success: Schema.Void,
  error: Schema.String,
  execute: Effect.gen(function*() {
    // Do the work...
    yield* Effect.log("Reserving inventory")
  })
})
```

Use the activity inside a workflow body by `yield*`-ing it:

```typescript
const OrderWorkflow = Workflow.make("OrderWorkflow", {
  payload: { orderId: Schema.String },
  idempotencyKey: ({ orderId }) => orderId
})

const OrderWorkflowLayer = OrderWorkflow.toLayer(({ orderId }) =>
  Effect.gen(function*() {
    yield* reserveInventory
    yield* Effect.log(`Reserved inventory for ${orderId}`)
  })
)
```

`Activity.retry` re-runs an activity while updating its
`Activity.CurrentAttempt`, and `Activity.idempotencyKey` derives a deterministic
key from the current execution ID and a name.

## Durable sleeps

`DurableClock.sleep` waits inside a workflow. Short durations run in memory; long
durations are persisted and resumed when the timer fires:

```typescript
import { DurableClock } from "effect/unstable/workflow"

yield* DurableClock.sleep({
  name: "wait-for-payment",
  duration: "24 hours"
})
```

Each sleep needs a stable `name` so the engine can correlate it with the
persisted timer on resume.

## Durable signals with DurableDeferred

To pause a workflow until an external event arrives, use `DurableDeferred`. The
deferred is awaited inside the workflow and completed from the outside (or from
another workflow):

```typescript
import { Effect, Schema } from "effect"
import { DurableDeferred, Workflow } from "effect/unstable/workflow"

const ApprovalDeferred = DurableDeferred.make("approval", {
  success: Schema.Boolean
})

const ApprovalWorkflow = Workflow.make("ApprovalWorkflow", {
  payload: { orderId: Schema.String },
  success: Schema.String,
  idempotencyKey: ({ orderId }) => orderId
})

const ApprovalWorkflowLayer = ApprovalWorkflow.toLayer(({ orderId }) =>
  Effect.gen(function*() {
    const approved = yield* DurableDeferred.await(ApprovalDeferred)
    return approved ? `${orderId}:approved` : `${orderId}:rejected`
  })
)

// From outside the workflow (e.g. an HTTP handler), complete the deferred:
const approve = (executionId: string) =>
  Effect.gen(function*() {
    const token = yield* DurableDeferred.tokenFromExecutionId(
      ApprovalDeferred,
      executionId
    )
    yield* DurableDeferred.succeed(ApprovalDeferred, { token, value: true })
  })
```

## Resuming and interrupting

A suspended workflow can be resumed or interrupted by execution ID:

```typescript
yield* IncrementWorkflow.resume(executionId)
yield* IncrementWorkflow.interrupt(executionId)
```

The `suspendedRetrySchedule` option on `Workflow.make` controls the schedule
used when retrying a suspended workflow.

## Complete example

```typescript
import { Effect, Layer, Schema } from "effect"
import {
  Activity,
  DurableClock,
  Workflow,
  WorkflowEngine
} from "effect/unstable/workflow"

const ProcessOrder = Workflow.make("ProcessOrder", {
  payload: {
    orderId: Schema.String,
    items: Schema.Array(Schema.String)
  },
  success: Schema.Struct({
    orderId: Schema.String,
    status: Schema.String
  }),
  idempotencyKey: ({ orderId }) => orderId
})

const reserveInventory = Activity.make({
  name: "reserveInventory",
  execute: Effect.log("Reserving inventory")
})

const chargePayment = Activity.make({
  name: "chargePayment",
  execute: Effect.log("Charging payment")
})

const shipOrder = Activity.make({
  name: "shipOrder",
  execute: Effect.log("Shipping order")
})

const ProcessOrderLayer = ProcessOrder.toLayer((payload) =>
  Effect.gen(function*() {
    yield* reserveInventory
    yield* chargePayment

    // Wait one day before shipping
    yield* DurableClock.sleep({
      name: "fulfillment-delay",
      duration: "1 day"
    })

    yield* shipOrder

    return { orderId: payload.orderId, status: "shipped" }
  })
)

const program = Effect.gen(function*() {
  const result = yield* ProcessOrder.execute({
    orderId: "order-456",
    items: ["item1", "item2"]
  })
  yield* Effect.log(`Order ${result.orderId} ${result.status}`)
}).pipe(
  Effect.provide(ProcessOrderLayer.pipe(
    Layer.provideMerge(WorkflowEngine.layerMemory)
  ))
)
```

## Use cases

- **Order processing** — multi-step order fulfillment
- **Approval flows** — wait for human approval in business processes
- **Scheduled tasks** — run tasks at specific times or intervals
- **Sagas** — coordinate distributed transactions
- **Event-driven processes** — react to events over long periods
